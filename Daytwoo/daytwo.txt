DAY 2 - 

 Git workflow summary (short)

1. `git init` — local repo banate ho.
2. File bana/modify karo → `git add` → staging area.
3. `git commit` — snapshot save karo.
4. Remote (GitHub) connect karo → `git push` — code upload.
5. Dusre ka code lene ke liye `git clone` (pehli baar) ya `git pull` (existing repo) use karo.

---

 1) `git init`

Kya karta hai: Current folder ko Git repository (local) bana deta hai. `.git` folder create hota hai jisme Git history store hoti hai.
Use:

```bash
cd project-folder
git init
```

Example: `git init` chalane ke baad `git status` se pata chalega repo ready hai.

---

 2) `git status`

Kya karta hai: Batata hai kaunse files changed hain, kaunse staged hain, aur current branch kaunsa hai. Bahut useful command.
Use:

```bash
git status
```

Output dekho: Untracked files (nayi files), Changes not staged, Changes to be committed (staged).

---

 3) `git add file.txt`  (Staging area)

Kya karta hai: File ko staging area mein daalta hai — matlab next commit mein include karne ke liye mark karta hai.
Use:

```bash
git add file.txt
```

Note: Agar multiple files add karne ho: `git add file1 file2` ya saare changes staging ke liye: `git add .` (current folder ke saare tracked/untracked files).

---

 4) `git add .`

Kya karta hai: Current directory ke sabhi changed aur new files ko staging mein daal deta hai. Simple aur common.

---

 5) `git commit -m "your message"`

Kya karta hai: Staged changes ko local repository mein ek snapshot (commit) ke form mein save karta hai. Message mein short description do.
Use:

```bash
git commit -m "Added login page and fixed CSS"
```

Tip: Commit messages meaningful rakhो — later samajhne mein easy hota hai.

---

 6) `git remote add origin <github-url>`

Kya karta hai: Local repo ko ek remote server (jaise GitHub) se connect karta hai aur us remote ka naam `origin` rakhta hai.
Use:

```bash
git remote add origin https://github.com/username/repo.git
```

Check: `git remote -v` se remote URL dekho.

Agar remote already set hai aur change karna ho:

```bash
git remote set-url origin https://github.com/username/new-repo.git
```

---

 7) `git push -u origin BranchName`

Kya karta hai: Local branch ke commits remote (origin) pe bhej deta hai. `-u` option upstream set karta hai (aage `git push` aur `git pull` bina origin/branch mention kiye chal sakte hain).
Use (pehli baar push karte waqt):

```bash
git push -u origin main
```

ya agar branch `master` ho to `git push -u origin master`.

Phir aage se: sirf `git push` chalana kaafi hota hai (agar upstream set ho).

---

 8) `git push`

Kya karta hai: Jo current branch ka upstream set hai, us remote pe commits bhejta hai.
Use:

```bash
git push
```

(agar upstream set nahi hai toh error aayega — tab `git push -u origin BranchName` use karo)

---

 9) `git clone <github-url>`

Kya karta hai: Remote repository ko poore local machine pe copy (clone) karta hai — isme repository history bhi aati hai. Use jab pehli baar kisi repo ko local karna ho.
Use:

```bash
git clone https://github.com/username/repo.git
```

Result: Ek folder `repo` create hoga jiski andar `.git` bhi rahegi.

---

 10) `git pull`

Kya karta hai: Remote repository se latest changes fetch karke aapke current branch mein merge (ya rebase) karta hai — basically remote changes laata hai aur local branch update karta hai.
Use (simple):

```bash
git pull
```

(agar upstream set hai)
Ya explicitly:

```bash
git pull origin main
```

Difference fetch vs pull:

 `git fetch` — remote changes get kar leta hai par merge nahi karta. (changes ko dekhte ho)
 `git pull` — fetch + merge (ya fetch + rebase) — remote changes ko local branch mein apply karta hai.

---

 Extra useful commands & tips

 Dekho current branch:

  ```bash
  git branch
  ```

  Current branch pe `` hota hai.

 Branch create & switch:

  ```bash
  git branch new-feature
  git checkout new-feature
   OR modern command
  git switch -c new-feature    creates and switches
  ```

 Commit history dekho:

  ```bash
  git log --oneline
  ```

 User config (agar pehli baar):

  ```bash
  git config --global user.name "Ayush Sharma"
  git config --global user.email "youremail@example.com"
  ```

 Undo staging:

  ```bash
  git reset HEAD file.txt
  ```

  (staging area se hata dega, file changes rehengi)

 Discard local changes in file:

  ```bash
  git checkout -- file.txt    WARNING: local changes lost
  ```

 Resolve merge conflicts: Agar `git pull` karte waqt conflict aaye to files manually edit karo, phir:

  ```bash
  git add conflicted-file
  git commit -m "Resolved merge conflict"
  ```

---

 Common errors & fixes

 Error: failed to push some refs → remote pe naye commits hain. Solution: `git pull` pehle karo, conflicts solve karo, phir `git push`.
 No upstream branch → use `git push -u origin BranchName`.
 Permission denied (publickey) → SSH key issue; temporary solution: use HTTPS URL or set up SSH keys.

 Typical Day-1 simple workflow (example)

1. `mkdir myproject && cd myproject`
2. `git init`
3. create `index.html`
4. `git status`
5. `git add index.html` or `git add .`
6. `git commit -m "Add index.html"`
7. `git remote add origin https://github.com/username/myproject.git`
8. `git push -u origin main`

Aage jab remote pe aur updates aayen:
`git pull` → kam karo → `git add .` → `git commit -m "..."` → `git push`
